## 概述

### 目的
在不改变现有页面结构与业务流程的前提下，将应用的主要页面统一到下载的 Figma 代码包视觉风格（emerald/teal 渐变 + amber 点缀），确保整体一致性与完成度。

### 范围
**包含**：
- 底部四 Tab 页面：首页（Capture）、社区（列表+详情）、历史、我的
- 分析流程页面：OCR、文本确认、分析中、Summary、Detail
- 账号与引导页面：登录、注册、Onboarding
- 底部导航样式统一（按 Figma 风格扩展为 4 Tab）

**不包含**：
- 后端接口、数据结构与业务流程变更
- 文案与页面结构重排
- 字体替换（保持现有字体）
- 新功能页面或路由新增

  - HomePage/ScanPage/ResultsPage/ProcessingPage 作为风格基准

## 功能需求

### 需求 1：统一基础视觉风格（Scoped 主题）
- **编号**: FR-029-001
- **优先级**: 高
- **描述**: 在页面级别应用 Figma 风格主题，保持结构不变，仅替换视觉风格。
- **验收标准**:
  - [ ] 背景为浅色渐变（emerald/teal → white）
  - [ ] 卡片为半透明白底 + 轻量模糊 + 圆角 24-28px + 阴影
  - [ ] 主按钮为 emerald/teal 渐变（含 hover/active 状态）
  - [ ] 次级按钮为浅色边框 + 轻量背景
  - [ ] 字体保持现有字体（不替换）

### 需求 2：底部导航对齐 Figma 风格
- **编号**: FR-029-002
- **优先级**: 高
- **描述**: 底部导航样式对齐 Figma，扩展为 4 个 Tab。
- **验收标准**:
  - [ ] Tab 顺序为“首页 / 历史 / 社区 / 我的”
  - [ ] 激活态带有圆角高亮背景与轻微发光
  - [ ] 非激活态灰度降低、可点击反馈明显
  - [ ] 视觉风格与 Figma 代码包一致

### 需求 3：分析流程页面视觉统一
- **编号**: FR-029-003
- **优先级**: 高
- **描述**: OCR、确认、分析中、Summary、Detail 统一使用 Figma 风格视觉组件。
- **验收标准**:
  - [ ] 顶部栏/标题/按钮与 Figma 风格一致
  - [ ] 结果卡片、列表、标签、徽章的样式一致
  - [ ] Loading/等待状态具备 Figma 风格的卡片与进度表现

### 需求 4：账号与引导页面视觉统一
- **编号**: FR-029-004
- **优先级**: 中
- **描述**: 登录、注册、Onboarding 页面使用 Figma 风格主题，保持结构与文案不变。
- **验收标准**:
  - [ ] 输入框、按钮与卡片视觉统一
  - [ ] 页面背景与品牌区风格统一

### 需求 5：社区与历史页面视觉统一
- **编号**: FR-029-005
- **优先级**: 中
- **描述**: 社区列表/详情与历史页面卡片、列表与按钮对齐 Figma 风格。
- **验收标准**:
  - [ ] 列表卡片具备统一圆角、阴影与边框样式
  - [ ] 分页与操作按钮样式统一

## 非功能需求

### 可用性
- 样式改动不影响现有交互与数据展示
- 触摸区域与可读性保持

### 性能
- 避免引入重型依赖或额外图片资源
- CSS 改动不显著增加渲染负担

## 成功指标

- 主要页面视觉风格一致且符合 Figma 参考
- 底部导航风格统一且可用
