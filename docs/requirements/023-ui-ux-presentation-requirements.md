# UI 展示优化需求说明

## 背景与目标
当前结果页已具备可信度与规则命中，但信息层级与决策引导仍偏“堆叠式”。本次优化聚焦三点：信息分层、可视化与对比、决策引导，提升用户理解速度与分享动力。

## 目标范围
- 信息分层：先结论后证据，降低认知负担。
- 可视化与对比：用更直观的图形表达风险与可信度。
- 决策引导：针对人群与场景给出明确建议。

## 非目标
- 不引入复杂 BI 或外部对比数据源。
- 不新增多语言与主题系统。

## 功能需求
1) 信息分层
- 结果页顶部显示“结论卡”（可吃/谨慎/不推荐）+ 一句话理由。
- 关键风险成分以 3–5 个标签突出展示。
- 规则明细与证据保持折叠，默认收起。

2) 可视化与对比
- 可信度展示为进度条/刻度条，并显示关键因子。
- 风险维度可视化：糖/添加剂/过敏原/钠等维度用条形或雷达图展示。
- 若无对比数据，展示“同类对比暂无”占位，不影响主流程。

3) 决策引导
- 针对选择的人群（过敏/儿童/孕妇/控糖等）生成专属提醒卡。
- 提供“可替代建议”卡片（基于规则库与判定，优先给出方向性建议）。

## 交互与文案
- 默认隐藏复杂明细，允许用户逐层展开。
- 强提醒仍保持显眼样式。
- 文案风格专业可信，避免营销化措辞。

## 验证标准
- UI 层级清晰，用户首屏可获取结论与关键风险。
- 可信度可视化与风险维度图可正常渲染。
- 人群建议与替代建议卡片出现且与偏好一致。
- 前端 `cargo check` 通过（允许 warnings），服务端流程完整通过。
