## 目标
以最小成本完成移动端静态 UI，覆盖“拍照 → 分析中 → 结果表格”三页结构，便于后续接入模型接口与状态管理。

## 范围
- 包含：页面结构、路由跳转、静态文案、基础样式、占位数据。
- 不包含：相机调用、图片上传、接口联调、数据持久化、权限控制。

## 技术选型
- 框架：Leptos（CSR）
- 宿主：Tauri（后续接入）
- 样式：纯 CSS（模块化 class 命名）
- 路由：leptos_router

## 页面与路由

| 页面 | 路由 | 说明 |
|------|------|------|
| 拍照页 | `/` | 入口页，包含拍照按钮 |
| 分析中页 | `/analyzing` | 进度占位 |
| 结果页 | `/result` | 配料表表格 |

## 组件结构
- `App`：全局布局与路由容器
- `CapturePage`：标题 + 说明 + 拍照按钮
- `AnalyzingPage`：加载占位 + 文案
- `ResultPage`：结果表格 + 重新拍照按钮
- `IngredientTable`：表格组件（静态数据）

## UI 布局与样式
- 竖屏优先，容器宽度 360-420px 视窗适配。
- 主按钮使用高对比色，尺寸不小于 48px 高度。
- 表格列：成分名称 / 风险等级 / 说明，支持多行文本。
- 颜色与字号：统一变量定义在 `app.css`，方便后续调整。

## 静态数据约定
结果页表格使用固定占位数据，字段如下：
- `name`：成分名称
- `risk`：低 / 中 / 高
- `note`：简短说明

## 目录结构建议
```
frontend/src/
  main.rs
  lib.rs
  pages/
    mod.rs
    capture.rs
    analyzing.rs
    result.rs
  components/
    mod.rs
    ingredient_table.rs
  styles/
    app.css
```

## 开发步骤
1) 建立路由与页面占位。
2) 搭建三页静态布局与通用按钮样式。
3) 添加结果页表格组件与占位数据。
4) 调整移动端间距与字体。

## 风险与备注
- 路由与样式结构需为后续接入相机与接口预留扩展点。
- 若后续采用不同 UI 风格，可在 `app.css` 统一替换主题变量。
