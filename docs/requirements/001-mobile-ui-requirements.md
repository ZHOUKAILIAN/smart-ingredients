## 概述

### 目的
定义一个极简移动端 UI，用于“拍照 → 分析 → 展示配料表”的静态页面结构。

### 范围
仅包含页面结构与静态 UI，不涉及接口、状态管理与业务逻辑。

## 功能需求

### 需求 1：拍照页
- **编号**: FR-001
- **优先级**: 高
- **描述**: 展示一个明显的拍照按钮作为主入口。
- **验收标准**:
  - [ ] 页面包含标题与简短说明
  - [ ] 页面包含一个“拍照”主按钮

### 需求 2：分析中页
- **编号**: FR-002
- **优先级**: 中
- **描述**: 展示分析中的静态状态。
- **验收标准**:
  - [ ] 居中展示加载占位
  - [ ] 文案为“正在分析配料表…”

### 需求 3：结果页
- **编号**: FR-003
- **优先级**: 高
- **描述**: 展示配料表分析结果的静态 UI。
- **验收标准**:
  - [ ] 显示“配料表”标题
  - [ ] 显示表格结构（名称、风险等级、说明三列）
  - [ ] 提供“重新拍照”按钮

## 非功能需求

### 性能
- 静态 UI 首屏渲染不超过 2 秒（开发环境可忽略）。

## 约束条件

### 技术约束
- 移动端竖屏布局。

### 业务约束
- 仅静态 UI，不接入模型或接口。

## 依赖关系

### 外部依赖
- 无。

## 成功指标

- 三个页面可静态浏览并通过视觉验收。
- 拍照入口清晰，结果页表格结构完整。
