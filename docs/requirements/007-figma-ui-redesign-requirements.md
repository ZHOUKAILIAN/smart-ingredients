## 概述

### 目的
根据 Figma 站点原型（Smart Ingredients）统一 App 首页、拍摄上传页、分析报告页的 UI 与交互表现，提升视觉一致性与体验完成度。

### 范围

**包含内容**：
- 首页（Home）布局、文案、视觉样式
- 拍摄/上传页（Scan）空态与已上传状态
- 分析报告页（Results）结构与信息层级
- 三页之间的交互流转与按钮状态

**不包含内容**：
- OCR、LLM 等算法逻辑优化
- 新的业务功能或数据字段
- 后端 API 变更

### 利益相关者
- **终端用户**：获得更清晰一致的 UI 和更顺畅的操作体验
- **产品团队**：确保设计稿落地一致性
- **前端开发**：需要完成 UI 与交互重构

## 功能需求

### 需求 1：首页 UI 结构与样式对齐 Figma

- **编号**: FR-007-001
- **优先级**: 高
- **描述**:
  - 背景采用浅色渐变（emerald/teal 到 white），纵向布局
  - 顶部品牌区包含应用图标、AI 角标、标题、Slogan 与简短说明
  - “使用步骤”以卡片形式展示 3 步，包含序号圆角标、图标、标题、描述
  - “查看示例”以轻量文本按钮呈现
  - 底部提供主 CTA “开始分析”按钮
  - 模块顺序为：品牌区 → 使用步骤卡 → 查看示例 → 主 CTA

- **验收标准**:
  - [ ] 首页布局与 Figma 版式一致，包含品牌区、步骤卡、示例入口与 CTA
  - [ ] 颜色、圆角、阴影、字号层级与 Figma 视觉一致
  - [ ] “开始分析”按钮为主操作，具备明显视觉权重
  - [ ] 标题与文案与 Figma 完全一致（Smart Ingredients / AI智能配料表分析 / 拍摄识别配料表，AI分析健康风险，让您吃得更安心）

### 需求 2：拍摄/上传页交互与状态

- **编号**: FR-007-002
- **优先级**: 高
- **描述**:
  - 顶部包含返回按钮与页面标题“拍摄配料表”
  - 空态展示：上传引导区、主按钮“拍照”、次按钮“从相册选择”、拍摄小贴士
  - 已上传状态：展示图片预览、删除按钮、提示文案“✓ 图片已上传，点击分析开始识别”、主按钮“开始分析”
  - 分析进行时：按钮变为“AI 分析中...”并显示旋转加载图标

- **验收标准**:
  - [ ] 空态与已上传态 UI 与 Figma 一致
  - [ ] 上传后可清除图片并回到空态
  - [ ] 分析中按钮禁用，并展示加载状态与文案
  - [ ] 拍摄小贴士包含三条说明，顺序与文案一致

### 需求 3：分析报告页结构与信息层级

- **编号**: FR-007-003
- **优先级**: 高
- **描述**:
  - 顶部含返回与分享按钮，标题“分析报告”
  - 健康评分卡片展示评分数字、评价文案与进度条
  - 配料分析列表包含风险徽章（安全/注意/风险）与描述
  - 营养成分列表展示数值与百分比条，并包含脚注说明
  - 健康建议以列表卡片呈现
  - 底部固定操作栏包含“返回首页”“分析新产品”

- **验收标准**:
  - [ ] 报告页所有区块齐全，布局与视觉层级符合 Figma
  - [ ] 风险等级徽章颜色与文案对应低/中/高三档
  - [ ] 底部操作栏固定在视口底部，不遮挡主内容

### 需求 4：页面流转与导航逻辑

- **编号**: FR-007-004
- **优先级**: 中
- **描述**:
  - 首页点击“开始分析”进入拍摄/上传页
  - 上传成功并点击“开始分析”进入分析报告页
  - 分析报告页“返回首页”与顶部返回均回到首页
  - “分析新产品”返回拍摄/上传页空态

- **验收标准**:
  - [ ] 页面流转与按钮行为符合设计稿交互路径
  - [ ] 返回逻辑保持一致，避免出现悬空状态

### 需求 5：等待页面统一风格

- **编号**: FR-007-005
- **优先级**: 中
- **描述**:
  - OCR 识别等待页与 LLM 分析等待页使用统一渐变背景和卡片式布局
  - 等待页展示进度条与两行文案（标题 + 时长提示）
  - 错误态按钮样式与首页 CTA 一致（主/次按钮）

- **验收标准**:
  - [ ] OCR 等待页视觉样式与 Figma 首页一致
  - [ ] 分析等待页视觉样式与 Figma 首页一致
  - [ ] 等待页文案清晰区分识别阶段与分析阶段

## 非功能需求

### 性能
- 页面切换与按钮反馈在 200ms 内完成视觉响应

### 可用性
- 在 320px-430px 宽度范围内可用且布局不溢出

### 一致性
- 使用统一的按钮、卡片、徽章样式与圆角规范

## 用户故事

### 故事 1
- **作为** 新用户
- **我想要** 在首页快速了解功能并开始分析
- **以便** 立即体验产品价值

### 故事 2
- **作为** 复用用户
- **我想要** 快速上传并得到报告，同时能再次分析新产品
- **以便** 频繁进行配料检查

## 用例

### 用例 1：拍照并查看报告
- **参与者**: 用户
- **前置条件**: App 正常启动
- **主流程**:
  1. 在首页点击“开始分析”
  2. 拍照或从相册选择一张配料表图片
  3. 点击“开始分析”
  4. 进入分析报告页查看结果
- **后置条件**: 报告页展示分析结果与建议
- **备选流程**: 上传后删除图片，返回空态重新选择

## 约束条件

### 技术约束
- 仅调整前端 UI 与交互，不新增后端依赖

### 业务约束
- 保持现有功能流程不变，仅替换视觉与交互表现

## 依赖关系

### 内部依赖
- 现有 UI 组件库（按钮、卡片、徽章、进度条）

### 外部依赖
- Figma 站点原型作为视觉标准

## 成功指标

- 页面转化路径（首页 → 报告页）无阻塞
- 关键入口按钮点击率提升或保持稳定
