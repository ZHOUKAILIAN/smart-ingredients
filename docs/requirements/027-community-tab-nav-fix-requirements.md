## 概述

### 目的
修复底部导航未覆盖“社区”Tab 的编译问题，并确保导航行为与社区需求一致。

### 范围
**包含**：
- 底部导航 Tab 列表补全“社区”
- Tab 路由识别与上次访问路径记录覆盖社区
- Tab 图标渲染补全社区图标

**不包含**：
- 社区页面 UI 或功能改动
- 后端接口与数据结构变更

### 利益相关者
- 终端用户：可以从底部导航进入社区
- 前端开发：保证编译通过与导航一致性

## 功能需求

### 需求 1：底部导航补全社区 Tab
- **编号**: FR-027-001
- **优先级**: 高
- **描述**: 底部导航按“首页 / 历史 / 社区 / 我的”的顺序展示四个 Tab。
- **验收标准**:
  - [ ] 底部导航包含“社区”Tab
  - [ ] Tab 顺序为“首页 / 历史 / 社区 / 我的”

### 需求 2：社区 Tab 路由识别与路径记忆
- **编号**: FR-027-002
- **优先级**: 高
- **描述**: 路由识别与上次访问路径记录覆盖社区路径。
- **验收标准**:
  - [ ] `/community` 与 `/community/*` 路由识别为社区 Tab
  - [ ] 切换 Tab 时可记忆并返回社区的上次访问路径

### 需求 3：社区 Tab 图标渲染
- **编号**: FR-027-003
- **优先级**: 中
- **描述**: 底部导航渲染社区 Tab 图标。
- **验收标准**:
  - [ ] 社区 Tab 使用现有图标组件渲染
