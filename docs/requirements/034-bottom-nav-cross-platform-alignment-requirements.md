## 背景

当前底部导航在浏览器与 Android WebView/Tauri 中视觉表现不一致：
- 图标与文字组合在底部栏内出现下沉，未保持垂直居中。
- 页面内容区与底部栏预留高度不一致，造成视觉错位。

## 目标

统一底部导航在 Web 与 Android 端的布局行为，确保图标、文字和可点击区域一致且居中。

## 功能需求

- FR-001：底部导航每个 Tab 的图标与标题在各自点击区域中垂直居中。
- FR-002：底部栏容器与内容区底部预留高度统一，避免出现额外偏移。
- FR-003：保留现有四个 Tab（首页、历史、社区、我的）与导航逻辑，不改交互语义。
- FR-004：Tab 图标尺寸保持一致，点击热区满足移动端最小触控要求。

## 验收标准

- AC-001：Android 模拟器中底部栏图标+文字与浏览器版本视觉位置一致。
- AC-002：底部栏所有 Tab 的图标、文字均居中对齐，无某一项偏上/偏下。
- AC-003：页面内容不会被底部栏遮挡，也不会留下不必要的额外空白。
- AC-004：`cargo check` 通过。

## 非目标

- 不调整图标形状、颜色主题。
- 不改动 Tab 导航路由逻辑。
